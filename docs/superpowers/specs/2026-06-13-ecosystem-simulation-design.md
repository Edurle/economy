# 生态模拟游戏 - 设计文档

> 日期：2026-06-13
> 状态：已批准，待实施

## 概述

一个基于 Python + pygame 的生态模拟沙盒游戏。玩家以上帝视角观察和干预一个包含多种地形、动物和自然现象的生态系统，通过调整参数和直接干预来维持或打破生态平衡。

## 技术选型

- **语言**：Python 3.10+
- **图形库**：pygame-ce（pygame 社区版，性能更好）
- **数值计算**：numpy（地形/草地用数组操作，高性能）
- **地形生成**：noise 库（柏林噪声生成自然地形）
- **架构**：ECS（Entity-Component-System）

## 世界设定

- **网格大小**：200 x 200 = 40,000 格
- **视图**：全局缩略图，整个地图缩放到窗口内，每格 1~3 像素
- **时间模型**：Tick 离散步进，速度可调（1x/2x/5x/10x），可暂停/单步

## 地形系统

共 6 种地形，每格固定为一种：

| 地形 | 草生长 | 动物可进入 | 能量消耗 | 水分消耗 | 特殊效果 |
|------|--------|-----------|---------|---------|---------|
| 草地 (GRASSLAND) | 正常 | 是 | 正常 | 正常 | 无 |
| 森林 (FOREST) | x1.5 | 是（大型动物减速） | 正常 | 正常 | 小动物获隐蔽，天敌发现概率 -50% |
| 水 (WATER) | 否 | 否 | — | — | 相邻格动物可饮水补充水分 |
| 沙漠 (DESERT) | 否 | 是 | 正常 | x2.0 | 无草无水，脱水死亡地带 |
| 山地 (MOUNTAIN) | 否 | 否（鹿除外） | — | — | 天然屏障，只有鹿能穿越 |
| 雪地 (SNOW) | 否 | 是 | x1.5 | x1.5 | 冬季时从地图边缘扩散 |

### 地图生成

使用柏林噪声（Perlin Noise）生成自然地形分布：
- 水聚集在低洼处形成湖泊和河流
- 沙漠成片出现在干旱区域
- 森林成簇分布
- 山地形成山脉走向
- 草地填充其余区域
- 雪地初始不存在，冬季时从地图边缘（北方）向内扩散

### 草系统

每个草地/森林格子有 `grass_level`（0~100）：
- 每 tick 自然恢复（基础 `growth_rate`，受季节和降雨影响）
- 灌木（森林中的植物）有独立的 `grass_level`，营养价值更高
- 羊/兔子/鹿吃草时消耗 `grass_level`，转化为自身能量
- 草被吃光后从 0 逐渐恢复

## 动物系统

### 食物链

```
草 / 灌木                    ← 生产者
    ↑
兔子 / 羊 / 鹿               ← 初级消费者（食草）
    ↑
狐狸                          ← 次级消费者（吃兔子、羊幼崽）
    ↑
狼                            ← 顶级捕食者（吃羊、鹿、狐狸）
```

### 通用组件

每只动物拥有以下属性（ECS 组件）：

| 属性 | 说明 |
|------|------|
| `energy` | 能量值（0~max_energy），吃食物恢复，每 tick 消耗，归零则死亡 |
| `hydration` | 水分值（0~max_hydration），饮水恢复，沙漠/雪地加速消耗，归零则死亡 |
| `age` | 年龄，每 tick +1，超过 `max_age` 自然死亡 |
| `position` | 网格坐标 (x, y) |
| `state` | 行为状态：FORAGING / FLEEING / HUNTING / MATING / SEEKING_WATER / IDLE |
| `target` | 目标实体 ID（猎物/配偶/水源方向） |
| `reproduce_cooldown` | 繁殖冷却剩余 tick |
| `diseased` | 是否感染瘟疫 |

### 物种参数

| 物种 | 初始能量 | 最大能量 | 最大水分 | 寿命(max_age) | 视野范围 | 移动速度(格/tick) | 繁殖条件 | 繁殖冷却 | 食物 | 天敌 | 全局数量上限 |
|------|---------|---------|---------|--------------|---------|-----------------|---------|---------|------|------|------------|
| 兔子 RABBIT | 30 | 60 | 50 | 25 | 3 | 2 | 能量>40 | 5 | 草(补充10能量) | 狐狸、狼 | 800 |
| 羊 SHEEP | 50 | 100 | 70 | 50 | 4 | 1 | 能量>70 | 10 | 草/灌木(补充15) | 狼 | 500 |
| 鹿 DEER | 80 | 150 | 90 | 70 | 5 | 1 | 能量>100 | 15 | 灌木/草(补充25) | 狼(需相邻2只以上) | 300 |
| 狐狸 FOX | 50 | 90 | 60 | 45 | 5 | 1 | 能量>60 | 12 | 兔子、羊幼崽 | 狼 | 300 |
| 狼 WOLF | 70 | 130 | 80 | 60 | 6 | 1 | 能量>90 | 20 | 羊、鹿、狐狸 | 无 | 200 |

### 行为 AI（优先级从高到低）

每只动物每 tick 按以下优先级决策，选定后设置 `state` 和 `target`：

1. **水分 < 25%？** → SEEKING_WATER：寻找最近水源，朝其方向移动到相邻格饮水
2. **天敌在视野内？** → FLEEING：朝远离天敌方向逃跑（猎物行为：兔子、羊、鹿、狐狸）
3. **猎物在视野内？** → HUNTING：追逐最近的猎物（捕食者行为：狐狸、狼）
4. **能量 < 40%？** → FORAGING：寻找最近的草地/猎物觅食
5. **满足繁殖条件？** → MATING：寻找同类求偶繁殖
6. **以上都不满足** → IDLE：随机游走

### 移动规则

- 移动方向为上下左右四方向（不允许斜向）
- 不可进入水格（除非 SEEKING_WATER 到达水边）
- 不可进入山地（鹿除外）
- 森林中大型动物（鹿、狼）移动速度 -1（最低 1）
- 兔子每 tick 可移动 2 格（速度快）

### 捕猎规则

- 捕食者与猎物相邻时发起攻击
- **捕猎成功率**取决于物种组合：
  - 狼吃羊：70%
  - 狼吃鹿：需 2 只以上狼相邻（鹿体型大），成功后每只狼分得能量
  - 狼吃狐狸：60%
  - 狐狸吃兔子：65%
  - 狐狸吃羊幼崽（age < 5 的羊）：90%
- 捕猎成功：猎物死亡，捕食者能量补充（猎物的能量值 x 转化率 0.8）
- 捕猎失败：双方都消耗少量能量，猎物可能触发 FLEEING

### 繁殖规则

- 两只同种动物相邻 + 都满足繁殖条件（能量达标、冷却结束）→ 生成幼崽
- 幼崽出生在父母相邻的空位
- 幼崽初始能量 = 父母各分出 20% 能量
- 出生后父母进入繁殖冷却期
- 受季节影响：春季繁殖概率 x1.2
- 全局数量上限：达到上限后不再繁殖（防止种群爆炸）

## 自然现象

### 季节系统

每 50 tick 切换一个季节，春夏秋冬循环：

| 季节 | 草生长率 | 降雨概率 | 动物能量消耗 | 水分消耗 | 特殊效果 |
|------|---------|---------|------------|---------|---------|
| 春 SPRING | x2.0 | 中高 | 正常 | 正常 | 繁殖概率 x1.2（繁殖季） |
| 夏 SUMMER | x1.0 | 最高 | 正常 | 正常 | 火灾风险升高 |
| 秋 AUTUMN | x0.5 | 低 | x1.2 | 正常 | 草生长减缓，动物积极觅食 |
| 冬 WINTER | x0（草停止生长，每 tick -1 直到归零） | 低（降雪） | x2.0 | 正常 | 雪地从地图边缘向内扩散，水源可能结冰（50% 概率不可饮） |

### 降雨系统

**状态机**：正常 → [随机触发] → 降雨(持续 5~15 tick) → 正常

- 长期无雨（50+ tick）→ 进入干旱状态
- **降雨效果**：受影响格子草地 `grass_level` 每 tick +5
- **干旱效果**：草地每 tick -3，持续干旱导致草地退化为沙漠
- 降雨概率受季节影响：夏季最高，冬季最低（冬季为降雪）
- 降雨为区域性的：以随机点为中心半径 15~30 格的圆形区域

### 火灾系统

- **触发条件**：夏季 + 连续 20 tick 无雨 + 草地 `grass_level > 80`
- **触发概率**：满足条件时每 tick 每格 0.1% 概率自燃
- **蔓延规则**：着火的格子每 tick 向相邻草地扩散，蔓延概率 40%（受降雨抑制）
- **效果**：
  - 烧毁该格草（`grass_level` 归零）
  - 该格动物 50% 概率烧死
  - 火持续 3~5 tick 后熄灭
- **灰烬 buff**：火烧过后，该格获得 `ash_buff`（30 tick），期间草生长 x3（自然界火循环促进重生）

### 瘟疫系统

- **触发条件**：某物种数量达到全局上限的 90%
- **传播**：每 tick 感染个体有 15% 概率传染给相邻同类
- **效果**：被感染个体每 tick 能量 -5（持续衰减）
- **结束条件**：该物种数量降至全局上限的 40% 后瘟疫消退（不再新发感染，已感染者逐渐恢复或死亡）
- **意义**：核心的自我调节机制，防止单一物种失控

## 上帝模式交互

### 玩家工具

| 工具 | 操作 | 说明 |
|------|------|------|
| 放置动物 | 选择物种后点击/拖拽地图 | 放置选中的动物（兔子/羊/鹿/狐狸/狼） |
| 改变地形 | 选择地形后点击/拖拽地图 | 将格子改为草地/森林/水/沙漠/山地/雪地 |

| 引火 | 点击草地 | 在点击位置引发火灾 |

| 查看 | 鼠标悬停 | 显示格子详情：地形、草量、动物及其状态 |

## 技术架构（ECS）

### 架构总览

- **Entity**：纯整数 ID，无逻辑
- **Component**：纯数据（dataclass），按类型存储在字典中
- **System**：纯逻辑，每 tick 按固定顺序执行

### 组件定义

```
Position:       x, y
Vitality:       energy, hydration, age
Species:        kind (RABBIT / SHEEP / DEER / FOX / WOLF)
Behavior:       state (FORAGING / FLEEING / HUNTING / MATING / SEEKING_WATER / IDLE), target_entity_id
Reproduction:   cooldown, can_breed
Health:         diseased, sick_timer
```

### 系统执行顺序（每 tick）

```
 1. SeasonSystem      — 切换季节，更新全局环境参数
 2. WeatherSystem     — 降雨/干旱判定，更新 rain_map
 3. GrassSystem       — 草生长（受季节、降雨、灰烬 buff 影响）
 4. FireSystem        — 火灾触发与蔓延
 5. PlagueSystem      — 瘟疫触发与传播
 6. AISystem          — 动物决策（按优先级选 state + target）
 7. MovementSystem    — 根据 state 移动动物
 8. ForagingSystem    — 食草动物吃草
 9. PredationSystem   — 捕食者捕猎
10. HydrationSystem   — 饮水补充水分 / 水分消耗
11. ReproductionSystem — 繁殖判定
12. AgingSystem       — 年龄+1、能量水分消耗、瘟疫伤害、死亡判定
13. CleanupSystem     — 移除死亡实体
```

### 世界数据结构

```
World:
  # 网格地形（numpy 数组）
  terrain:        np.ndarray (200, 200), int8   # TerrainType 枚举
  grass_level:    np.ndarray (200, 200), float32
  ash_buff:       np.ndarray (200, 200), int16   # 灰烬 buff 剩余 tick
  fire_map:       np.ndarray (200, 200), int8    # 燃烧剩余 tick，0=未燃烧
  rain_map:       np.ndarray (200, 200), bool_   # 当前是否降雨
  snow_edge:      int                              # 雪地扩散到的行数（冬季）

  # ECS 核心
  entities:       set[int]
  components:     dict[type, dict[int, Component]]
  next_id:        int

  # 全局状态
  season:         Season
  season_timer:   int            # 本季节剩余 tick
  drought_timer:  int            # 连续无雨 tick 数
  tick:           int
  speed:          int            # 1/2/5/10
  paused:         bool

  # 统计
  population_history: dict[SpeciesKind, deque]  # 用于趋势图
```

### 渲染策略

窗口布局（约 1200x800）：

```
┌──────────────────────────────────┬──────────────┐
│                                  │  数据面板     │
│        地图区域 (全局缩略图)       │              │
│        200x200 缩放到 ~800x800    │  各物种数量   │
│        每格 = 4 像素              │  [趋势曲线图] │
│        地形为背景色               │              │
│        动物为亮点（按物种着色）     │  [工具栏]     │
│                                  │              │
│                                  │  [参数滑块]   │
├──────────────────────────────────┤              │
│  状态栏：季节/tick/速度/事件提示   │              │
└──────────────────────────────────┴──────────────┘
```

- **地图渲染**：用 `pygame.surfarray` 把 numpy 颜色数组直接映射为 Surface 像素，再 `smoothscale` 缩放，极快
- **动物渲染**：在地图 Surface 上叠加像素点（每物种不同颜色）
- **趋势图**：用 `collections.deque` 维护最近 500 tick 的各物种数量，用 `pygame.draw` 画折线
- **帧率分离**：模拟 tick 与渲染帧率解耦；10x 速度时每帧执行 10 个 tick

## 验收标准

1. 世界启动后自动运行，可见狼追羊、羊吃草的自然行为
2. 6 种地形正确生成和显示，柏林噪声产生自然的分布
3. 5 种动物各自遵循食物链，能繁殖、捕猎、死亡
4. 季节循环可见效果（冬季草枯萎、动物死亡增多）
5. 降雨、火灾、瘟疫可自然触发并有视觉效果
6. 上帝模式所有工具可用（放置动物、改地形、降雨、引火、瘟疫、清除、查看）
7. 数据面板实时显示各物种数量和趋势曲线
8. 速度调节（1x/2x/5x/10x）和暂停/单步正常工作
9. 200x200 地图在 10x 速度下保持流畅（目标 30+ FPS）
10. 参数滑块实时生效，能观察到对生态的影响
